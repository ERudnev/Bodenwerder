  intendation matters (Python-style)
  
  Q1 language specification
  keywords:
    'namespace' : opens named namespace
    'declare' : declares existing external type
    'type' : defines new type or alias
    'template' : similar to C++ template
    '<...>' : similar to C++ template parameter list
    'struct' : defines structural type
    'as' : similar to "typedef/using in C++"
    'enum'
    '~' : type-of operator (reference to the type of an expression/member)
    '::' : namespace scope or global namespace if starts path
    ':' : "has type" (for structure field)
    '@' : preprocessor directives (seebelow)
    'entity' : declares entity (root particle)
    'attribute' : declares attribute (particle with Owner)
    'component' : declares component (mandatory particle with Owner)
    'resource' : declares external runtime-bound resource (binding)
    'anchor' : marks anchor reference (special persistent link semantics)
    'anchor(any)' : used for container of anchors with rule: "i exist if at least anchor valid"
    'anchor(all)' : used for container of anchors with rule: "i exist if all anchors valid"
    'depends' : adds dependency manually
    'element' : opens per-item data+operations block inside aspect definition
    'table' : opens field-own data+operations block inside aspect definition

    '?' : declares read-only operation (query) signature
    '*' : declares state-changing operation (command) signature
    '+' : declares creation factory (convenience create) signature
    '!' : declares validator/invariant signature
    '->' : declares operation return type

  reserved:
    'id' - autogenerated subtype
    'composite' - subjective and scope-dependent "soft" type for building composites of entities/attributes/components
    'particle'
    'model' : opens "model" block (domain model declaration)

  preprocessor directives:
    @builtin : this type definition is internal to the Q1 language
    @anchor : this is reference to Anchor entity

  resource declarations:
    `resource R @external(X)` : declares resource aspect R bound to opaque external runtime type X.
      X is NOT a Q1 type. It is an external/runtime identifier resolved by projection layer.
      Q1 declaration does NOT define concrete loader/binding implementation for X.
      Such runtime code is expected to exist outside DSL projection.

  buildtin types:
    'integer' : fixed size (32 bit) integer
    'float' : floating-point number
    'string' : UTF-8 string
    'boolean' : boolean (true/false)

    'optional<T>' : optional value container
    'vector<T>' : sequence container
    'map<K,V>' : ordered key-value container
    'set<K>' : ordered key container
    'umap<K,V>' : unordered key-value container
    'uset<T>' : unordered key container
  
    'seconds' : relative time duration (in seconds)
    'time' : timepoint

    'vec3' : glm-compatible 3-floats vec
    'quat' : glm-compatible 4-floats quaternion

  particle blocks:
    'element' : per-item state+queries/commands (stored in Field container as Item Quantum)
    'table' : field-own state+queries/commands (Global + fieldwide ops)
