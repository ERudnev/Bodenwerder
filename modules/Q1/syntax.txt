  intendation matters (Python-style)

  Q1 language specification
  keywords:
    'import' : line at top of file; pulls in another `.q1.types` module (see directives below)
    'namespace' : opens named namespace
    'declare' : declares existing external type
    'using' : defines new type or alias
    'template' : similar to C++ template
    '<...>' : similar to C++ template parameter list
    'struct' : defines structural type (body: local type aliases, fields, and optionally operation lines `?` `=` `>` — not `!`)
    'as' : introduces alias target in `using A as B` (when B is a type_expr; for type-of use `using A as ~Expr`)
    'enum'
    '~' : type-of operator (reference to the type of an expression/member)
    '::' : namespace scope or global namespace if starts path
    ':' : "has type" (for structure field: `field: Type` and for named parameters like `name: Type`)
    '->' : operation return type after the closing `)` (for example `name(args) -> ReturnType`). If `ReturnType` is the local aspect id written as `#`, use spaces around the arrow: `-> #`.
    '@' : preprocessor directives (seebelow)
    'entity' : declares entity (root particle)
    'attribute' : declares attribute (particle with Owner)
    'component' : declares component (mandatory particle with Owner)
    'agent' : declares external runtime-bound agent (wrapper/binding to an external resource)
    'depends' : adds dependency manually
    'one' : opens per-item data+operations block inside aspect definition
    'all' : opens field-own data+operations block inside aspect definition
    'always' : opens static constants + pure functions block inside aspect definition
    '#' : id type constructor (inside type expressions only):  #T == T::id (decode only) ; # == id (local Aspect type; decode only)
    '?' : postfix Optional type constructor (inside type expressions only):  T? == optional<T> (decode only)

  function prefix symbols:
    '?' : declares read-only operation (query) signature
    '=' : declares modifier limited with scope of local item (for "one") this Type item set (for "all")
    '>' : declares effector: may change object state or not, but changes world state. Facrics are effectors (returning '#' or '#T')
    '!' : declares reaction (normalizator) signature. Has no return parameter. Has no arguments, but defined scope (see section below)

  function scope symbols:
    '-' : "T::!foo(-one)" reacts on one T element deletion
    '=' : "T::!foo(=one)" reacts on one T element value changed

  reserved:
    'composite' - subjective and scope-dependent "soft" type for building composites of entities/attributes/components
    'particle'
    'model' : opens "model" block (domain model declaration)

  directives:
    @builtin : this type definition is internal to the Q1 language

    line-oriented import (processed before Q1 parse):
      `import "..."` : bring another `.q1.types` module into scope. The string is a logical path within the module (commonly rooted at that module's `aspects/` tree). Do not repeat the `.q1.types` suffix in the string.


  agent declarations:
    `agent R @external(X)` : declares an agent aspect R bound to an opaque external runtime **resource** type X.
      X is NOT a Q1 type. It is an external/runtime **resource** identifier resolved by projection layer.
      Q1 declaration does NOT define concrete loader/binding implementation for X.
      Such runtime code is expected to exist outside DSL projection.
    C++ projection note: operations from `element`/`table` on an agent aspect use the same `Reading`/`Writing`/`Id` context as other aspects; the generator does not add `resources::Manager` (or similar) to those operation signatures — runtime access goes through the world-bound provider (`modules/Q1/q1-to-iQSM.tome`, declaration_forms agent).

  buildtin types:
    'integer' : fixed size (32 bit) integer
    'index2' : 2D signed integer pair (x/y)
    'float' : floating-point number
    'string' : UTF-8 string
    'boolean' : boolean (true/false)
    'vector<T>' : sequence container
    'map<K,V>' : ordered key-value container
    'set<K>' : ordered key container
    'umap<K,V>' : unordered key-value container
    'uset<T>' : unordered key container

    'seconds' : relative time duration (in seconds)
    'time' : timepoint

    'mat4' : glm-compatible 4x4 float matrix
    'vec3' : glm-compatible 3-floats vec
    'quat' : glm-compatible 4-floats quaternion

  struct bodies (inside `struct`, alongside fields):
    Operation signatures use the same markers as in aspect `element`, but `!` (validator/invariant) is not allowed.
    `?` query, `=` command, `>` factory with explicit return type (for a value-struct factory, typically `-> StructName`)

  particle blocks:
    'element' : per-item state+queries/commands (stored in Field container as Item Quantum)
    'table' : field-own state+queries/commands (Global + fieldwide ops)
    'static' : compile-time constants + referentially transparent functions (no World/Global/Id dependency)
