# FQSM — Flat Quantized State Mechanism

**FQSM** (или **fQSM**) расшифровывается как **Flat Quantized State Mechanism**.

* **Flat** — плоское состояние без встроенного версионирования.
* **Quantized State** — состояние, представленное набором типизированных квантов данных.
* **Mechanism** — механизм исполнения, изменения и согласования такого состояния.

## Overview

FQSM представляет собой execution mechanism для DSL-описанных моделей мира.

DSL определяет структуру предметной области: типы, данные, действия, реакции и правила их взаимодействия.

FQSM предоставляет набор C++ механизмов, необходимых для исполнения этих определений внутри приложения.

## Responsibilities

FQSM определяет:

* представление квантизованного состояния;
* способы доступа к состоянию;
* механизмы изменения состояния;
* применение и интеграцию изменений;
* правила согласования состояния;
* исполнение реакций и действий;
* инфраструктуру исполнения пользовательской логики.

## Architecture

В типичном сценарии DSL используется для описания модели предметной области.

На основании этого описания генерируется C++ код, использующий типы и механизмы FQSM.

Во время выполнения приложения FQSM обеспечивает работу полученной модели и управление её состоянием.

## Design Goal

Цель FQSM состоит в предоставлении единого механизма исполнения для структурных и функциональных моделей, описанных на DSL и реализованных в C++.
