# Вычислимый мир и интерпретатор

Сегодня неожиданно сложились две давно существовавшие идеи.

Первая — детерминированная модель мира.

Мир описывается состоянием (`State`). Операторы изменяют его (`Patch`). Интеграция создаёт новое состояние. Изменения можно представить как язык событий (`Events`).

```text
State
  ↓
Actions
  ↓
Patch
  ↓
Integration
  ↓
New State
  ↓
Events
```

Это обычная инженерная модель. Она тестируема, воспроизводима и полностью детерминирована.

Вторая идея — использование LLM.

До сегодняшнего дня казалось естественным использовать LLM как автора, который придумывает историю.

Сегодня оказалось интереснее другое.

LLM вообще не обязана придумывать мир.

Она может **интерпретировать уже вычисленный мир**.

Например, такой лог:

```text
UNIT_ATTACKED attacker=1 target=2 damage=2
UNIT_ATTACKED attacker=2 target=1 damage=1
UNIT_ATTACKED attacker=1 target=2 damage=2
UNIT_DIED unit=2
...
```

оказался достаточным, чтобы LLM без знания внутренних алгоритмов модели написала полноценную сцену боя.

Она ничего не вычисляла.

Она ничего не придумывала о физике мира.

Она лишь восстановила человеческое восприятие уже произошедших событий.

Получается неожиданное разделение ответственности.

```
CPU отвечает за истину.

LLM отвечает за смысл.
```

Или иначе:

```
CPU вычисляет.

LLM рассказывает.
```

Причём рассказ — лишь одна из возможных интерпретаций.

Один и тот же журнал событий может стать:

* главой романа;
* сценарием кат-сцены;
* военным рапортом;
* дневником участника;
* репликами NPC;
* комментарием матча;
* объяснением игроку, почему произошло событие.

Модель мира остаётся неизменной.

Меняется только интерпретатор.

Следующий шаг выглядит ещё интереснее.

LLM может стать не только рассказчиком, но и одним из участников вычислений.

```
State
    ↓
CPU
    ↓
Events
    ↓
LLM
    ↓
Command
    ↓
CPU
```

Таким образом возникает цикл, в котором:

* мир вычисляется детерминированно;
* LLM принимает решения или объясняет происходящее;
* вычислительная модель остаётся единственным источником истины.

Это отличается от идеи "генерировать роман нейросетью".

Роман становится не источником событий, а **проекцией вычисленного мира**.

---

## Что меня удивило

Удивительно не то, что LLM умеет писать.

Удивительно то, насколько маленьким оказался необходимый интерфейс между вычислением и языком.

Тринадцать строк событий оказалось достаточно, чтобы возникла сцена.

Не потому, что модель языка придумала хороший сюжет.

А потому, что **сюжет уже существовал внутри причинной модели мира**.

LLM лишь позволила человеку его увидеть.

---

И последнее, уже почти личное.

Сегодня мне показалось, что языки моделирования могут оказаться нужны не только для вычислений.

Они могут стать **общим языком между алгоритмами и языковыми моделями**.

Не заменяя вычисления.

А делая вычисленный мир понятным человеку.
